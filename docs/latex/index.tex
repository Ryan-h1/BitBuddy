 \doxylink{class_bit_buddy}{Bit\+Buddy} is a silly little C++ game developed in the Qt framework.\hypertarget{index_autotoc_md1}{}\doxysubsection{\texorpdfstring{Dependencies}{Dependencies}}\label{index_autotoc_md1}

\begin{DoxyItemize}
\item A modern C++ compiler (Opening and running the project in CLion should be enough)
\item Qt 6.\+6.\+1 or later
\item CMake 3.\+26 or later
\item A Unix environment (this is because the project assumes the use of libc++ or libstdc++ for the standard library implementation)
\item CMake
\item A build system supported by CMake (We recommend using Ninja and use it in our example)
\end{DoxyItemize}\hypertarget{index_autotoc_md2}{}\doxysubsection{\texorpdfstring{Running the Program}{Running the Program}}\label{index_autotoc_md2}
You need to ensure that CMake can find your QT 6 installation. You can set this manually in the {\ttfamily CMake\+Lists.\+txt} by appending a manual{\ttfamily CMAKE\+\_\+\+PREFIX\+\_\+\+PATH}, you can set the {\ttfamily CMAKE\+\_\+\+PREFIX\+\_\+\+PATH} environment variable to the path of your Qt 6 installation, or you can specify the path to the Qt 6 installation when running CMake via the command line.

We recommend using CLion\textquotesingle{}s built in CMake support to build and run the project; however, if you would like to build the project manually, you can do so by running the following commands from the root directory of the project (from the directory that this README.\+md file is in)\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{cmake\ -\/DCMAKE\_PREFIX\_PATH=/path/to/Qt/6.6.3/your\_platform\ -\/S\ <source\_dir>\ -\/B\ <build\_dir>\ -\/G\ <generator>}
\DoxyCodeLine{cd\ <build\_dir>}
\DoxyCodeLine{cmake\ -\/-\/build\ .\ -\/-\/config\ Release}

\end{DoxyCode}

There should now be a \textquotesingle{}bitbuddy\textquotesingle{} executable in the build directory. You can run this executable to play the game.\hypertarget{index_autotoc_md4}{}\doxysubsection{\texorpdfstring{Structure of the Project}{Structure of the Project}}\label{index_autotoc_md4}
We decided to separate header files from source files into the {\ttfamily include} and {\ttfamily src} directories, respectively. This allows readers to navigate through only the include directory to see the public interface of the project, but they can still switch to the implementation of a header file with a hotkey (usually F10).
